# BBS Markdown 渲染器升级总结

我们对 BBS 的 Markdown 渲染引擎进行了深度定制，使其渲染能力向 GitHub/语雀等顶级平台看齐，不仅彻底解决了之前的代码块问题，还加入了现代化的设计元素。

## 🚀 升级内容

### 1. 流程图可视化 (`Mermaid` 支持)
- 引入了业界标准的 `mermaid` 渲染库。
- 我们不再把 `language="mermaid"` 的区块当做普通的文本代码块渲染，而是拦截它并动态将其转化为高清矢量图 (SVG)。
- **深色模式适配**：图表默认采用 `dark` 主题，完美契合 BBS 极客风格的代码高亮背景。
- **容错处理**：即使你的 Mermaid 语法写错了，渲染器也不会导致页面白屏崩溃，而是会在图表区域优雅地显示出错误日志提示。

### 2. GitHub 风格提示框 (`Alerts` 完美还原)
通过对 AST（抽象语法树）的深度解析和 React 节点拦截重组，我们为 BBS 加入了原汁原味的 GitHub Alerts 扩展语法支持：
- `> [!NOTE]` 👉 **蓝色信息框** (搭配 Ant Design 的 Info 图标)
- `> [!TIP]` 👉 **绿色提示框** (搭配 Check 图标)
- `> [!IMPORTANT]` 👉 **紫色重要框** (搭配 Exclamation 图标)
- `> [!WARNING]` 👉 **黄色警告框** (搭配 Warning 图标)
- `> [!CAUTION]` 👉 **红色高危框** (搭配 Close 图标)

不仅自动剔除了原有的 `[!xxx]` 标记字符，还在左侧添加了醒目的色彩边界，极大地提升了技术文章的阅读体验和重点信息的传达效率。
