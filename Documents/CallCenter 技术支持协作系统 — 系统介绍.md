## 一、系统概述

CallCenter 是一套 **IT 技术支持工单协作平台**，专为二线技术支持团队设计，提供工单管理、实时协作、知识沉淀和 AI 辅助等能力。

### 核心定位

| 维度 | 说明 |
|-----|------|
| **使用场景** | 客户技术问题的接单、处理、协作、归档全流程 |
| **目标用户** | 一线工程师（开单）、二线技术支持（处理）、技术总监（管理）、外部客户（协作） |
| **部署方式** | 私有化部署，数据自主可控 |
| **访问方式** | Web 浏览器（PC / 移动端自适应），未来支持独立 APP |

## 四、安全体系

### 4.1 身份认证

| 安全措施 | 说明 |
|---------|------|
| **JWT 双令牌机制** | Access Token（短期，15分钟）+ Refresh Token（长期，7天，HttpOnly Cookie） |
| **密码加密** | bcrypt 哈希算法（10轮加盐），数据库不存储明文密码 |
| **Token 刷新** | Access Token 过期后通过 Refresh Token 无感续期 |
| **WebSocket 认证** | WebSocket 连接时验证 JWT Token，无效立即断开 |
| **外部用户隔离** | 外部共享链接生成独立 Token，仅能访问指定工单 |

### 4.2 权限控制（RBAC）

系统采用 **基于角色的访问控制（RBAC）** 模型：

**预设角色：**

| 角色 | 说明 | 权限范围 |
|-----|------|---------|
| `admin` | 超级管理员 | 拥有所有权限（全局通行） |
| `director` | 技术总监 | 管理 + 报表 + 高级操作 |
| `tech` | 二线技术支持 | 工单处理 + 接单 + 知识库 |
| `user` | 普通用户（一线） | 创建工单 + 基本查看 |
| `external` | 外部共享用户 | 仅限指定工单的聊天 |

**支持自定义角色**，可灵活配置权限组合。

**细粒度权限点（19项）：**

| 资源 | 权限 | 说明 |
|-----|------|------|
| tickets | read / create / delete / assign / edit / share | 工单相关 |
| knowledge | read / manage / generate / export_history | 知识库相关 |
| admin | access | 后台管理 |
| report | read | 数据报表 |
| settings | read / edit | 系统设置 |
| bbs | read / create / edit / delete / comment | 论坛相关 |

**权限执行机制：**
- 每个 API 接口通过 `@Permissions()` 装饰器声明所需权限
- 三层守卫链路：`JwtAuthGuard → RolesGuard → PermissionsGuard`
- 管理员自动通行，外部用户仅允许访问显式标记的接口

### 4.3 审计日志

| 功能 | 说明 |
|-----|------|
| **用户登录审计** | 记录每次登录（成功/失败）、IP 地址、时间 |
| **外部用户接入审计** | 记录外部用户通过共享链接接入的行为 |
| **工单状态变更审计** | 记录工单的接单、关单等状态流转 |
| **审计开关** | 各类型审计可独立开关，灵活控制 |
| **日志查询** | 支持按类型、关键字、时间范围筛选 |
| **日志清理** | 支持按条件批量清理历史日志 |

### 4.4 数据安全

| 安全措施 | 说明 |
|---------|------|
| **HTTPS 加密传输** | 所有通信均通过 SSL/TLS 加密 |
| **CORS 白名单** | 仅允许配置的域名跨域访问 |
| **请求验证** | 全局 ValidationPipe，自动校验请求参数，拒绝非法输入 |
| **SQL 注入防护** | 使用 TypeORM 参数化查询，杜绝 SQL 注入 |
| **XSS 防护** | 前端输入过滤 + 后端参数白名单验证 |
| **全局异常捕获** | 统一错误处理，不向客户端暴露内部堆栈信息 |
| **文件上传安全** | 文件存储于腾讯云 COS，不直接暴露服务器文件系统 |
| **Cookie 安全** | Refresh Token 使用 HttpOnly + SameSite 属性，防止 XSS/CSRF |
| **IP 记录** | 所有关键操作记录客户端 IP（通过 X-Forwarded-For 解析真实 IP） |

### 4.5 聊天室安全

| 安全措施 | 说明 |
|---------|------|
| **房间锁定机制** | 工单创建人/接单人可锁定聊天室，仅受邀人员可参与 |
| **外部链接禁用** | 可禁用外部共享链接，阻止外部用户进入 |
| **踢出用户** | 创建人/接单人/管理员可踢出未授权用户 |
| **消息定向投放** | 新消息通知仅推送给工单参与者，防止信息泄露 |
| **权限实时更新** | 角色变更时通过 WebSocket 广播，在线用户立即生效 |

## 六、信息部可能关心的问题 & 建议回答

### Q1：系统的数据存储在哪里？是否安全？

> **A：** 数据存储在我们自己管理的腾讯云服务器上（上海节点），数据库使用 MySQL 8.0，文件使用腾讯云 COS 对象存储。所有数据归公司所有，不经过任何第三方。通信全程 HTTPS 加密。

### Q2：系统的认证安全机制是什么？

> **A：** 使用 JWT 双令牌认证机制（Access Token 15分钟过期 + Refresh Token 7天），密码使用 bcrypt 10轮加盐哈希存储。对接 SSO 后，所有认证将通过公司统一认证平台完成，我们不再独立管理密码。

### Q3：系统有权限管理吗？如何控制不同人员的访问？

> **A：** 有完整的 RBAC 权限体系。预设 5 种角色（管理员/总监/二线/一线/外部），支持自定义角色。共 19 个细粒度权限点，每个 API 接口都有权限检查。

### Q4：系统有操作审计功能吗？

> **A：** 有。系统记录用户登录（含失败）、外部用户接入、工单状态变更等操作，每条记录包含操作人、IP 地址、时间、详细描述。支持查询和清理。

### Q5：系统能对接我们的统一登录吗？技术方案是什么？

> **A：** 可以。根据分析，贵司统一认证平台（yx.trustfar.cn）使用 OAuth 2.0 授权码模式。我们只需要获得分配的 Client ID 和几个接口地址，即可完成对接。对接后可关闭独立注册和密码登录，所有入口统一走 SSO。

### Q6：系统数据能备份吗？

> **A：** 可以。系统内置一键全量备份功能，生成 ZIP 文件可下载到本地。也支持一键恢复。

### Q7：系统并发承受能力如何？

> **A：** 当前服务器配置（2C2G）可支撑日常使用（几十人同时在线）。如果用户量增长，可通过升级服务器配置或水平扩展来应对。实时通信基于 WebSocket，连接效率高。

### Q8：外部客户通过共享链接访问，数据安全吗？

> **A：** 外部用户通过加密 Token 接入，只能看到被授权的那一个工单。有房间锁定机制可随时禁止外部人员进入。消息通知也是定向推送，不会泄露其他工单信息。

### Q9：OMM 工单推送的接口安全吗？

> **A：** 工单推送接口将使用独立的服务端认证机制（API Key 或 Service Token），与普通用户登录分离。只有持有正确密钥的系统才能调用。

### Q10：对接周期预估多久？

> **A：** 取决于公司信息部的响应速度。从拿到所需信息开始计算：
> - 域名证书配置：**1 天**
> - SSO 登录对接：**3-5 个工作日**
> - 用户同步：**2-3 个工作日**
> - 工单推送（我方接收端）：**2-3 个工作日**
> - 联调测试：**2-3 个工作日**
> - **总计约 2-3 周**（不含公司侧 OMM 推送开发时间）
