# CallCenter 智能客服系统中台对接与部署方案

## 1. 项目背景与目标

目前，CallCenter 智能客服系统已完成初期开发与局域网内测试（当前运行于内网 IP `192.168.50.51`）。为了实现**企业级的数据安全管控**、**统一身份认证（SSO）**以及**全面解锁基于 WebRTC 的音视频高级功能**，本系统计划正式接入公司的统一中台与企业微信生态。

本方案旨在明确系统对接的底层逻辑、规范网络与安全要求，并列出需公司 IT 部门/中台研发团队协助的前置配置清单，以确保对接过程高效、平滑。

---

## 2. 核心技术痛点与 HTTPS 硬性要求（重点）

由于 CallCenter 系统深度集成了**实时屏幕共享**与**多人语音通话**等高级客服功能，底层强依赖于现代浏览器的 WebRTC 技术栈（`navigator.mediaDevices` 等 API）。

根据 Chrome、Edge 等主流浏览器的底层安全策略，**所有音视频捕获与屏幕共享功能，仅允许在安全的 HTTPS 环境（Secure Context）下运行**。若系统运行于纯 HTTP 协议或纯 IP 地址下，浏览器将静默拦截屏幕共享请求，导致核心业务功能失效。

> **结论：** 申请内部正式域名并配置合规的 SSL 证书（HTTPS），不仅是中台安全的规范要求，更是本系统核心功能得以运行的**物理级硬性前提**。

---

## 3. IT / 中台部门协助清单（Pre-requisites）

为推进对接落地，需请公司网络与中台部门协助完成以下配置：

### 3.1 域名与网络解析配置
*   **需求**：分配一个专属的内部子域名（例如：`callcenter.trustfar.cn` 或 `helpdesk.trustfar.cn`）。
*   **指向**：通过公司内部 DNS，将该域名解析至系统服务器的内网 IP `192.168.50.51`。

### 3.2 SSL 证书下发与 HTTPS 配置
*   **需求**：针对上述分配的域名，由公司内部 Root CA 或外部证书机构签发配套的 SSL 证书。
*   **部署**：将证书提供给系统研发团队配置于 Nginx 服务器，或由 IT 部门在前端负载均衡/反向代理层（网关）统一配置 SSL 卸载，确保终端用户通过 `https://` 访问。

### 3.3 统一认证中心（SSO）应用注册
系统采用业界标准的 **OAuth 2.0 / OIDC** 协议进行单点登录对接，需在中台/企业微信管理后台完成以下操作：
1.  **创建应用**：在中台注册一个名为“智能客服”的应用入口。
2.  **配置回调（Redirect URI）**：将授权回调地址设置为：`https://[分配的域名]/api/auth/sso/callback`（具体路径可微调）。
3.  **发放凭证**：向 CallCenter 系统研发负责人提供以下授权凭证：
    *   `client_id` (或 AppID)
    *   `client_secret` (或 AppSecret)
4.  **授权数据范围（Scope）**：登录成功后，需允许系统获取该员工的基本信息（包括但不限于：`UserID`、真实姓名、所属部门、邮箱/手机号）。

## 4. CallCenter 系统研发侧改造计划

在获取上述参数后，CallCenter 系统团队将在 3 个工作日内完成以下系统侧的改造闭环：

### 4.1 认证拦截与回调闭环
*   将原有的“账号密码登录”页面改造为单点登录跳板。
*   用户未登录访问时，系统将通过携带 `client_id` 与 `response_type=code` 的参数重定向至公司中台登录页。
*   中台回调携带 `code` 返回后，系统后端自动拿 `code` 换取用户真实身份信息。

### 4.2 JIT (Just-In-Time) 动态用户供给与权限管理
*   **自动建号**：员工首次通过中台扫码进入系统时，若数据库中无该用户，系统将**自动无感为其创建账号**，避免了前期繁琐的通讯录导入工作。
*   **权限分配**：
    *   默认情况下，所有新进入的用户将自动获得 **“申请人（普通用户）”** 权限，即可发起工单。
    *   对于运维/技术支持团队，系统将根据中台传回的“所属部门”标识进行正则匹配自动赋予“工程师”角色，或由系统超级管理员在后台进行一次性手动提权。

### 4.3 离职与账号风控机制
*   **天然阻断**：由于全面接入中台 SSO，当员工离职或账号在中台被禁用时，其在中台层面的认证即刻失效。该员工将无法通过回调进入 CallCenter，从根源上保障了系统数据的安全隔离。
*   *(可选扩展)*：未来可接入企业微信“通讯录变更回调” Webhook，实现账号禁用状态的实时数据库级同步。

---

## 5. 预期上线与割接效果

1.  **用户体验**：员工打开公司内部平台，点击“办公应用”列表中的“智能客服”图标，**全程无需输入账号密码**，即可瞬间进入工作台页面。
2.  **数据安全**：所有信令和音视频数据全程在 `https://` 加密通道内网传输，防窃听、防篡改。
3.  **功能解锁**：屏幕共享、截图、富文本粘贴、以及规划中的多人语音调度功能得以在安全沙箱内顺畅运作。

> **附件/备注**：
> 研发技术栈：Node.js (NestJS) + React + Vite + 原生 WebRTC。
> 对接协议强兼容标准的 OAuth 2.0、SAML 2.0 及企业微信/钉钉免登体系。
