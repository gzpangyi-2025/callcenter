# CallCenter 系统对接实施全流程指南

> **文档目的：** 为项目管理和技术开发人员提供清晰、明确的端到端系统对接实施路径，明确双方在各阶段的职责分工。

---

## 第一阶段：前期准备与参数交换（基建阶段）

**目标：** 打通网络基础，完成各类安全凭证与接口文档的互换，为正式开发扫清障碍。

### 📋 CallCenter团队提供给公司信息部/OMM团队：
1. **服务器与域名信息**：内网 IP `172.31.0.22`，域名 `callcenter.trustfar.cn`。
2. **SSO 回调地址**：`https://callcenter.trustfar.cn/api/auth/sso/callback`（测试与生产环境共用此地址）。
3. **工单接收接口地址**：若采用 HTTP 直推方式，提供完整的 API 接口地址。

### 🏢 公司信息部/OMM团队提供给CallCenter团队：
1. **SSO 凭证**：分配给 CallCenter 应用的 `Client ID` 和 `Client Secret`。
2. **SSO 接口地图**：统一认证平台的授权页 URL、获取 Token URL 及获取 UserInfo URL。
3. **基建材料**：域名的 SSL 证书文件（.crt/.key），企业微信的 CorpID 等。
4. **工单推送策略**：确认采用 HTTP 接口直推，还是采用消息中间件（MQ）。若采用 MQ，需提供连接地址与账号密码。

---

## 第二阶段：单点登录（SSO）开发与实施

**目标：** 实现公司员工免密、无感登录 CallCenter 系统，并实现账号的自动注册。  
**主导方：** CallCenter团队

### 🏢 公司信息部/OMM团队职责（配置配合）：
1. 在统一认证系统后台，将 CallCenter 提供的“回调地址”加入安全信任白名单。

### 📋 CallCenter团队职责（核心开发）：
1. **开发拦截重定向（改造入口）**：开发全局拦截逻辑，当未登录用户尝试访问系统时，将其重定向至公司统一认证网页。
2. **开发回调接收口（接引通道）**：开发专用回调接口，接收企业微信扫码成功后带回来的授权码（Code）。
3. **实现无感自动注册（数据落地）**：
   - 后端使用 Code 向公司信息部/OMM团队换取用户信息（工号、姓名、部门等）。
   - 在 CallCenter 数据库中进行比对：**若用户不存在，则静默创建新账号**；若存在，则直接签发本地登录令牌（JWT）。
   - 将用户引导至 CallCenter 首页，完成闭环。

---

## 第三阶段：工单自动推送开发与实施

**目标：** 实现 OMM 系统生成的二线支持工单，能够实时、准确地推送到 CallCenter 系统中。  
**主导方：** 公司信息部/OMM团队

### 📋 CallCenter团队职责（建好“信箱”）：
1. **部署接收服务**：
   - **若为 HTTP 方式**：严格按文档规范开发好 `POST` 接收接口。
   - **若为 MQ 方式**：开发对应的消费者程序（Consumer），持续监听指定的队列。
2. **发放通信密钥**：为 OMM 系统生成一个专用的后台通信密钥（Service Token），用于安全鉴权。

### 🏢 公司信息部/OMM团队职责（核心开发）：
1. **系统埋点与触发**：在 OMM 系统内部设定规则逻辑，监听工单状态变化。
2. **数据打包**：一旦某个工单的状态变更为“二线支持”，立即触发代码，将该工单的标题、描述、发起人工号、处理人工号等核心字段打包为 JSON 格式。
3. **执行推送**：
   - 携带CallCenter团队发放的 Service Token，将数据包通过 HTTP 请求发送至CallCenter团队接口，或投递至指定的 MQ 队列。

---

## 第四阶段：环境联调测试与正式上线

**目标：** 双方在真实网络环境下验证链路连通性及数据准确性，最终切入生产环境。

### 联调测试步骤：
1. **SSO 登录测试（CallCenter团队主导）**：
   - 使用真实的测试员工账号，验证扫码流程是否顺畅。
   - 验证 CallCenter 是否成功拉取到用户信息，且账号能够被正确自动注册。
2. **工单推送测试（双方配合）**：
   - 公司信息部/OMM团队在 OMM 测试环境中模拟创建一张工单，并手动将其转为“二线支持”。
   - 观察 CallCenter 系统内是否实时生成了对应的新工单。
   - 重点核对：人员匹配（工号是否正确映射到了责任人）、内容是否乱码或截断。
3. **修复与上线**：
   - 根据测试过程中的报错日志调整字段或代码逻辑。
   - 验证无误后，统一切换为正式域名（`callcenter.trustfar.cn`），完成系统最终上线。
