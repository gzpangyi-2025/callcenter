# CallCenter 代码评审问题整理

审计日期：2026-05-03  
审计范围：本次文档整理来自当前代码评审中新增的 4 条重点发现，覆盖后端权限控制、WebSocket 聊天鉴权、文件访问控制，以及前端 Markdown/Mermaid 富文本渲染安全。

## 总体结论

当前项目功能链路较完整，但以下问题集中暴露出一个共同风险：鉴权逻辑没有在所有入口保持一致。HTTP 接口、WebSocket 事件、文件下载、搜索聚合和前端富文本渲染之间缺少统一的安全边界，生产上线前应优先修复。

| 编号 | 严重级别 | 问题标题 | 主要影响 |
| --- | --- | --- | --- |
| 1 | P0 | 外部分享 token 可访问共享对象之外的数据 | 外链访客可能探测非授权工单、帖子或搜索结果 |
| 2 | P0 | 未锁定聊天室缺少内部成员校验 | 内部用户可能通过 ticketId 加入非授权聊天室并读取历史消息 |
| 3 | P1 | 公共静态文件服务可暴露任意上传内容 | 上传文件可能被公开访问，且缺少类型限制和业务授权绑定 |
| 4 | P1 | Markdown 和 Mermaid 渲染是 XSS 高风险面 | 用户内容或 AI 内容可能通过链接、SVG、HTML 高亮片段进入 DOM |

## Finding 1：外部分享 token 可访问共享对象之外的数据

严重级别：P0  
涉及文件：`backend/src/modules/auth/permissions.guard.ts:27-33`  

### 问题描述

`PermissionsGuard` 对 `external` 角色做了特殊处理：只要目标接口声明了 `tickets:read` 或 `bbs:read` 权限，就直接放行。代码注释中假设 controller 会继续按 `ticketId` 或 `bbsId` 做二次校验，但实际并非所有接口都做了这层约束。

### 影响范围

外部分享 token 原本应该只允许访问被分享的单个工单或帖子。但当前设计下，外部用户可能调用列表、聚合、搜索等接口，探测共享对象之外的数据。

重点风险入口包括：

- `backend/src/modules/tickets/tickets.controller.ts`：工单列表、聚合、批量摘要、按工单号查询等接口。
- `backend/src/modules/bbs/bbs.controller.ts`：帖子列表、评论列表等接口。
- `backend/src/modules/search/search.controller.ts`：全站搜索接口使用 `tickets:read` 权限，可能被外部工单 token 调用。

### 整改建议

1. 不要在通用 `PermissionsGuard` 中泛化放行 `external` 用户。
2. 为外部分享 token 单独设计 `ExternalTicketGuard`、`ExternalBbsGuard` 或等价的上下文鉴权逻辑。
3. 所有外部可访问接口必须显式校验 token 中的 `ticketId` 或 `bbsId`。
4. 列表、聚合、搜索、评论、批量摘要接口默认不对外部 token 开放；如果确需开放，必须强制按共享对象收敛查询范围。
5. 增加回归测试：外部 token 访问非共享工单、帖子、搜索结果时必须返回 403。

## Finding 2：未锁定聊天室缺少内部成员校验

严重级别：P0  
涉及文件：`backend/src/modules/chat/chat.gateway.ts:213-252`  

### 问题描述

`joinRoom` 事件只对 `external` 用户校验 `allowedTicketId`。对于内部登录用户，只在房间锁定时调用 `roomService.isAuthorizedForRoom`。如果房间未锁定，内部用户只要知道或猜到 `ticketId`，就可以加入对应聊天室并获取 `messageHistory`。

同类问题也会影响发消息、屏幕共享、语音通话等 WebSocket 事件，因为这些事件的鉴权边界没有统一复用工单访问权限。

### 影响范围

如果业务上存在定向工单、仅接单人可见工单、客户敏感工单等场景，当前 WebSocket 入口可能绕过 HTTP 列表页的可见性控制。即使前端没有展示某工单，用户仍可能通过直接构造 `ticketId` 加入聊天室并读取历史消息。

### 整改建议

1. 抽象统一的 `TicketAccessService`，判断用户是否可访问指定工单。
2. `joinRoom`、`sendMessage`、屏幕共享、语音通话、历史消息拉取等事件全部调用同一套工单访问校验。
3. 房间是否锁定不应决定基础访问权限；锁定只能作为更严格的附加限制。
4. 对外部用户、普通内部用户、创建人、接单人、参与者、管理员分别补充 WebSocket 鉴权测试。
5. 对非法访问尝试记录审计日志，便于发现批量探测 `ticketId` 的行为。

## Finding 3：公共静态文件服务可暴露任意上传内容

严重级别：P1  
涉及文件：`backend/src/modules/files/files.controller.ts:68-87`  

### 问题描述

`files/static/:filename` 没有鉴权，任何人拿到文件名都可以访问。本地文件通过 `res.sendFile` 以内联方式返回，COS 文件则重定向到预签名地址。同时上传接口没有限制文件 MIME 类型或扩展名，也没有把文件与工单、帖子、上传人或权限上下文绑定。

### 影响范围

当前实现可能导致以下问题：

- 工单附件、帖子附件或截图被公开访问。
- 用户上传 SVG、HTML 等浏览器可执行或可解释内容后，形成存储型 XSS、内容伪造或钓鱼入口。
- 文件访问无法按业务对象授权，也无法追踪谁有权下载某个文件。
- `migration-stats` 和 `migrate` 仅要求登录，普通用户可能触发存储迁移类操作。

### 整改建议

1. 建立文件元数据表，记录 `filename`、`originalName`、`mimetype`、`ownerId`、`businessType`、`businessId`、`visibility`。
2. 文件下载接口必须鉴权，并根据业务对象调用对应访问控制逻辑。
3. 上传接口增加 MIME 类型和扩展名白名单；默认禁止 SVG、HTML、JS 等高风险类型。
4. 对非图片或不可信类型使用 `Content-Disposition: attachment`，避免浏览器 inline 执行。
5. 存储迁移接口增加 `admin:access` 或专用运维权限。

## Finding 4：Markdown 和 Mermaid 渲染是 XSS 高风险面

严重级别：P1  
涉及文件：

- `frontend/src/components/MarkdownViewer.tsx:61-64`
- `frontend/src/components/Mermaid.tsx`
- `frontend/src/pages/GlobalSearch/index.tsx`

### 问题描述

`MarkdownViewer` 通过 `urlTransform={(value) => value}` 放开了链接协议转换。`Mermaid` 使用 `securityLevel: 'loose'` 并通过 `dangerouslySetInnerHTML` 注入生成的 SVG。全站搜索页使用 `dangerouslySetInnerHTML` 渲染 Elasticsearch highlight 片段。

这些内容来源可能包括工单描述、聊天记录、BBS 帖子、知识库、AI 生成内容或搜索索引内容，均不应被视为可信 HTML。

### 影响范围

攻击者可能通过 Markdown 链接、图片 URL、Mermaid 图表、SVG 内容或搜索高亮片段构造恶意内容，在其他用户浏览工单、帖子、知识库或搜索结果时触发 XSS。

### 整改建议

1. 移除自定义 `urlTransform`，或实现明确的协议白名单，仅允许 `http`、`https`、`mailto`、相对路径等安全协议。
2. 引入 `rehype-sanitize` 或 DOMPurify，对 Markdown 产物进行统一消毒。
3. Mermaid 优先使用 `securityLevel: 'strict'`；如确需复杂交互，考虑 iframe 沙箱隔离。
4. Elasticsearch highlight 不直接使用 `dangerouslySetInnerHTML`，改为解析高亮标记后渲染受控 React 节点。
5. 增加 XSS 用例测试，覆盖 Markdown 链接、图片、Mermaid SVG、搜索高亮片段。

## 优先整改路线

1. 第一优先级：修复外部分享 token 的越权访问，补齐外部 token 负向测试。
2. 第二优先级：统一 HTTP 与 WebSocket 的工单访问校验，避免聊天入口绕过列表可见性。
3. 第三优先级：重构文件访问模型，先禁止高风险类型并收紧迁移接口权限。
4. 第四优先级：统一前端富文本消毒策略，消除 Markdown、Mermaid、搜索高亮的 DOM 注入风险。

## 验收建议

完成整改后，建议至少验证以下场景：

- 外部工单 token 只能访问对应工单详情，不能访问工单列表、聚合、批量摘要、全站搜索。
- 外部 BBS token 只能访问对应帖子详情，不能访问帖子列表或其他帖子评论。
- 普通内部用户不能通过 WebSocket 加入无权限工单房间，也不能发送消息或参与屏幕共享。
- 未授权用户不能直接访问任意上传文件。
- 上传 SVG、HTML、JS 等高风险文件被拒绝或只能作为附件下载。
- Markdown、Mermaid、搜索高亮中的恶意脚本不会执行。
