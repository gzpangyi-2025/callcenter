# 腾讯云 COS 对象存储迁移总结

已成功将后端的本地存储架构迁移至腾讯云 COS。这将带来更好的可用性并显著降低本地磁盘压力。

## 主要变更

1. **COS 基础集成**
   - 引入了 `cos-nodejs-sdk-v5`。
   - 创建了 `FilesService` 管理存储桶连接及核心的上传/下载功能。
   - 使用系统提供的子账号 AK/SK 接入了 `callcenter-1425043423` 存储桶。

2. **接口重构与优化**
   - 废弃了基于本地磁盘的 `ServeStaticModule`。
   - 重构了 `FilesController` 的上传和下载接口：
     - 文件上传现在会使用 `memoryStorage` 将文件缓冲至内存并流式推送到云端。
     - 下载与静态访问（图片加载）不再消耗服务器带宽，而是重定向至具有 1 小时时效的 COS 预签名链接 (Presigned URL)。并支持定制下载的文件名称和响应头。

3. **文档与报告生成**
   - 对依赖本地磁盘读取的 `TicketsExportService` (用于导出聊天记录/处理报告的 DOCX 及 ZIP 文件) 进行了底层替换。
   - 现在，生成报告时如果需要插入工单中的图片及附件，会直接从 COS 中把文件流拉取到内存中，再通过 docx.js 进行嵌入打包。

4. **消息撤回与论坛清理机制**
   - 修复了此前撤回工单聊天消息无法自动清理云端资源的问题。现在当用户撤回包含图片或附件的消息时，后端会同步调用 COS 接口，将相应的文件从对象存储中永久销毁，防止云端空间浪费。
   - 同步修复了 **BBS 论坛板块**。现在用户在论坛中删除帖子（单删或批量删除），系统除了清空帖子及下属评论数据，也会自动提取所有相关的对象文件，并向腾讯云发起真实的物理销毁请求。

5. **AI 知识库图片**
   - 之前由 Gemini API 生成的流程图会保存在本地的 `uploads/knowledge` 文件夹中。
   - 现在代码也已更新，会将 AI 生成的 base64 图像直接推送到云端对象存储中。

6. **系统备份机制优化**
   - 正如规划所述，附件和图片上云后，这部分数据由云端的高可用机制保障。
   - 现已重构了系统的 `BackupService`，在触发一键打包备份时**跳过所有多媒体附件**。
   - 备份功能现完全聚焦于 SQLite 核心业务数据的留存，这会使备份速度产生质的飞跃，且生成的 zip 压缩包体积将极为轻量化。

## 环境部署状态

- `npm install` 等依赖已经在本地与 `192.168.50.51` 远程生产环境完成。
- COS 凭证环境变量 (`COS_SECRET_ID`, `COS_SECRET_KEY`, `COS_BUCKET`, `COS_REGION`) 已安全注入两端的 `.env` 配置中。
- PM2 服务 `callcenter-backend` 已在本地及远程重启完毕。
