# 首云对象存储（S3 兼容）双端独立接入实施计划

**目标**：完全满足您提出的要求，使 **CallCenter** 和 **AI 协作模块 (Codex Worker)** 都能**独立**、**同时**配置并使用不同的存储提供商（腾讯云 COS 或 首云 S3 兼容），实现全环境对两套对象存储的无缝兼容。

## User Review Required

> [!IMPORTANT]
> - 在 CallCenter 中，目前前端使用的是腾讯云专用的 `cos-js-sdk-v5` 和 STS 临时密钥进行大文件分片直传。引入首云 S3 后，为了兼容前端直传，当系统使用首云时，后端将为其生成 **S3 Pre-signed URL (预签名直传链接)**，前端通过标准的 HTTP PUT 方式将文件直接上传到首云。该方式对于大多数文件传输非常高效稳定。
> - 在执行前，由于这涉及到两端环境的底层存储逻辑（`callcenter/backend/src/modules/files/files.service.ts` 和 `codex-worker/src/storage/storage.service.ts`）和前端 API (`api.ts`)，请您确认该直传逻辑的变更是否符合预期。

## Open Questions

> [!NOTE]
> 首云提供的 Endpoint 是公网地址还是内网地址？
> - 对于 `Codex Worker`（部署在东京腾讯云），如果首云和腾讯云不在同一内网，必须使用首云的**公网 Endpoint**。
> - 在后台的配置面板中，我会增加供您输入 Endpoint 的文本框，到时候直接填入即可。

## Proposed Changes

由于要求两个模块相互独立，配置页和底层逻辑都需要改造：

### 第一部分：CallCenter (系统基础模块) 支持首云 S3

#### [MODIFY] `callcenter/backend/package.json`
- 增加 `@aws-sdk/client-s3` 和 `@aws-sdk/s3-request-presigner` 依赖。

#### [MODIFY] `callcenter/backend/src/modules/files/files.service.ts`
- 修改存储服务的实现，同时支持 `cos` 和 `s3` 两种 provider。
- 增加 `getS3Instance()` 方法管理首云 S3 的连接。
- 当 provider 为 `s3` 时：
  - `uploadToCos` 使用 `PutObjectCommand`。
  - `getPresignedUrl` 和 `generateUploadCredentials` 使用 `getSignedUrl(GetObjectCommand)` 和 `getSignedUrl(PutObjectCommand)` 生成预签名链接。
  - `getStorageStats` 使用 `ListObjectsV2Command` 统计。

#### [MODIFY] `callcenter/frontend/src/services/api.ts`
- 修改前端的 `uploadFile` 逻辑：判断后端返回的 `provider`。
  - 如果是 `cos`，保留现有的 `cos-js-sdk-v5` STS 分片上传。
  - 如果是 `s3`，则直接使用 `axios.put(credentials.presignedUrl, file)` 完成前端直传。

#### [MODIFY] `callcenter/frontend/src/pages/Admin/components/StorageManageTab.tsx`
- 在系统文件存储设置中增加「首云 S3 兼容」选项。
- 动态显示 S3 配置项：`S3 Endpoint`、`Access Key`、`Secret Key`、`Bucket`、`Region`。

---

### 第二部分：AI 协作模块 (Codex Worker) 支持首云 S3

#### [MODIFY] `codex-worker/package.json`
- 同样增加 `@aws-sdk/client-s3` 和 `@aws-sdk/s3-request-presigner` 依赖。

#### [MODIFY] `codex-worker/src/storage/storage.service.ts`
- 改造存储方法，读取 `.env` 中的 `STORAGE_PROVIDER` (`tencent` 或 `s3`)。
- 当配置为 `s3` 时，映射所有的 COS SDK 调用为 AWS SDK S3 调用（上传、下载、列出文件、删除文件等）。

#### [MODIFY] `codex-worker/src/tasks/tasks.controller.ts` & `runtime-config.service.ts`
- 接收 `STORAGE_PROVIDER` 以及 S3 相关的五个配置参数，并持久化到 `.env` 中。

#### [MODIFY] `callcenter/backend/src/modules/settings/settings.controller.ts`
- 在 `saveCodexConfig` 中增加代理 S3 字段的处理逻辑，推送配置到 Worker。

#### [MODIFY] `callcenter/frontend/src/pages/Admin/components/CodexConfigTab.tsx`
- 在 AI 协作引擎配置面板中，同样增加「首云 S3 兼容」单选项和相应的 S3 配置输入框，做到与系统配置完全隔离。
